# Resumen: Dinámica de Sistemas Continuos

## 1. Introducción a la Dinámica de Sistemas Continuos

La **simulación de sistemas** se define como la técnica de resolver problemas siguiendo los cambios en el tiempo de un modelo dinámico de un sistema. Se utiliza cuando no es posible obtener una solución analítica.

- **Simulación continua**: Se utiliza cuando el interés principal está en cambios suaves, describiendo el sistema mediante conjuntos de **ecuaciones diferenciales**.
- La **Dinámica de Sistemas** es una disciplina que proporciona un enfoque estructurado para modelar, analizar y comprender sistemas complejos. Se basa en la teoría de sistemas y utiliza conceptos de **retroalimentación** y **causalidad**.

## 2. Definición de Sistemas Continuos

Los **sistemas continuos** son aquellos cuyas variables evolucionan de forma continua en el tiempo. Se describen mediante ecuaciones diferenciales ordinarias (EDO).

- **Ecuaciones de estado**:
  $$\begin{cases}
\dot{x}_1 = f_1(x_1, x_2, \dots, x_{n-1}, x_n, t) \\
\dot{x}_2 = f_2(x_1, x_2, \dots, x_{n-1}, x_n, t) \\
\vdots \\
\dot{x}_n = f_n(x_1, x_2, \dots, x_{n-1}, x_n, t)
\end{cases}$$
- **Ejemplo**: Evolución de la concentración de una droga en el estómago.
  $$c_e'(t) = -r_e c_e(t)$$

## 3. Solución Analítica de Sistemas Continuos

Se pueden resolver mediante métodos como:

- **Método de autovalores**: Diagonalización de la matriz de coeficientes.
- **Transformada de Laplace**: Convierte ecuaciones diferenciales en ecuaciones algebraicas, útil para sistemas lineales.

## 4. Definición de Dinámica de Sistemas

- **Según Jay Wright Forrester**: Estudio de la estructura de sistemas complejos y cómo cambian a lo largo del tiempo. Un sistema es un conjunto de elementos interrelacionados que interactúan para lograr un propósito.
- **Metodología**: Modelado y simulación para estudiar el comportamiento de sistemas complejos, basada en niveles, flujos, relaciones de retroalimentación y retardos.
- **Pasos para su estudio**:
  1.  Observar el comportamiento del sistema real.
  2.  Identificar las estructuras de realimentación.
  3.  Construir el modelo matemático.
  4.  Simular el modelo.
  5.  Modificar la estructura hasta que coincida con el comportamiento observado.
  6.  Modificar las decisiones para encontrar soluciones aceptables.
- **Herramientas principales**:
  - **Diagrama Causal o de Influencias**: Representa las relaciones cualitativas entre variables.
  - **Diagrama de Forrester o de Simulación**: Representación gráfica detallada que modela el comportamiento dinámico.

## 5. Diagrama Causal o de Influencia

Es una representación gráfica de las relaciones de causa y efecto entre los elementos de un sistema.

- **Representación**:
  - **Nodos**: Elementos o variables del sistema.
  - **Flechas** (`→`): Indican la dirección de la influencia.
  - **Signos** (`+` o `-`): Indican la naturaleza de la relación.
    - **Signo positivo (+)**: Las variaciones son del mismo sentido.
    - **Signo negativo (-)**: Las variaciones son de sentido contrario.
- **Clasificación según su estructura**:
  - **Abiertos**: No forman lazos cerrados.
  - **Cerrados o Bucles de Realimentación**: Contienen ciclos de interacción.

### i. Bucles de Realimentación

Una cadena cerrada de causas y efectos.

- **Signo del bucle**: Producto algebraico de los signos de las relaciones.
  - **Bucle Positivo (+)**: Número par de relaciones negativas. Refuerza el cambio (crecimiento o declive exponencial). *Ejemplo: Crecimiento poblacional.*
  - **Bucle Negativo (-)**: Número impar de relaciones negativas. Contrarresta el cambio, buscando estabilidad o equilibrio. *Ejemplo: Regulación de la temperatura.*
- **Bucles Combinados**: Coexisten bucles positivos y negativos. El comportamiento final depende del bucle dominante.

### ii. Interpretación de Influencias

- Cada flecha muestra cómo una variable afecta a otra.
- El signo revela si la influencia es de aumento o disminución.

### iii. Consideraciones Importantes

- Usar elementos cuantificables y variables que cambien en el tiempo.
- Evitar bucles ficticios, redundantes y el tiempo como factor causal.
- Los nombres de las variables deben ser sustantivos (ej. "Presas", no "Aumento de presas") y especificar un signo (ej. "Temperatura", no "FRÍO").

## 6. Diagrama de Forrester o de Simulación

Es un paso intermedio entre el diagrama causal y las ecuaciones diferenciales. Clasifica las variables en tres tipos.

### i. Símbolos Utilizados

1.  **Variable de Nivel (Estado)**:
    - **Símbolo**: Bloque integrador.
    - **Función**: Acumulan los resultados de acciones pasadas. Son las variables principales del sistema.
    - **Ecuación**: `Nivel = ∫ (Flujo) dt`
    - **Características**: Cambian lentamente, son unidades sin tiempo.
    - **Representación**: `[ ]` o `( )`

2.  **Variable de Flujo**:
    - **Símbolo**: Válvula.
    - **Función**: Determinan la variación de las variables de nivel (entradas y salidas).
    - **Ecuación**: `Flujo = f(Niveles, Auxiliares, Constantes)`
    - **Características**: Su unidad es `Unidad de Nivel / Tiempo`.
    - **Representación**: `⨁` o `→|`

3.  **Variable Auxiliar**:
    - **Símbolo**: Círculo.
    - **Función**: Representan pasos intermedios en el cálculo de los flujos. Convierten información para facilitar la definición de los flujos.
    - **Representación**: `○`

4.  **Fuente y Sumidero (Nubes)**:
    - **Función**: Representan el exterior del sistema con capacidad infinita. La fuente es el origen de un flujo, el sumidero es su destino.
    - **Representación**: `☁️`

5.  **Canales**:
    - **De Material**: Transportan flujos entre niveles, fuentes o sumideros.
    - **De Información**: Transportan información para alimentar variables auxiliares y flujos.

## 7. Construcción de un Diagrama Causal y de Simulación

**Ejemplo: Llenar un vaso de agua**

1.  **Identificación de Elementos**:
    - `Nivel de Agua` (Nivel)
    - `Nivel Deseado` (Auxiliar)
    - `Discrepancia` (Auxiliar)
    - `Flujo de Agua` (Flujo)
    - `Tasa de flujo de agua` (Constante)

2.  **Diagrama Causal**:
    - `Flujo de Agua` →+ `Nivel de Agua`
    - `Nivel de Agua` →+ `Discrepancia`
    - `Nivel Deseado` →- `Discrepancia`
    - `Discrepancia` →+ `Flujo de Agua`

3.  **Diagrama de Forrester**:
    - Se utiliza el símbolo de nivel para `Nivel de Agua`.
    - Se utiliza el símbolo de válvula para `Flujo de Agua`, regulado por la `Discrepancia`.
    - `Nivel Deseado` y `Tasa de flujo de agua` son constantes/auxiliares que influyen en la decisión del flujo.

4.  **Modelo Computacional (Ecuaciones)**:
    ```
    (01) Discrepancia = Nivel Deseado - Nivel de Agua
    (03) flujo de agua = Discrepancia * Tasa de flujo de agua
    (05) Nivel de Agua = INTEG(flujo de agua, 0)
    (06) Nivel Deseado = 250
    (08) Tasa de flujo de agua = 0.2
    ```
    El proceso se puede simular en software como Vensim.
